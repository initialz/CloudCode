5. Behavioural caveats

  • Visiting https://<hub>/app/ now serves the SPA shell from the /*spa fallback — React Router won't have a route for /app, so the user lands on a blank-ish page or the SPA's catch-all. Per the operator's explicit "no bookmark-compat" call, this is by design.
  • Existing webterm tabs still pointed at /app/* with cached /app/api/* URLs in their bundle will 404 on every API call until the user hard-refreshes. Cookie still rides along (Path=/), so once they hit / they're already logged in.
  • Did NOT test the SPA UI flows in a browser (multi-tool split, drag-select copy, Settings) — those are covered by the existing v1.11.x manual smoke plan and the path move shouldn't affect them.
  • Did NOT test the v1.11.x → v1.12.0 self-update path end-to-end. The supervisor + symlink-flip logic is unchanged; the only delta is the binary serves SPAs on different paths after the swap, which the OSC-52 / wire / sessions code doesn't touch.
