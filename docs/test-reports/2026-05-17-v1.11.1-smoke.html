4. Hub self-update — under supervisor PASS

Setup

Reset the sandbox state, this time spawning the v1.11.0 hub via the supervise subcommand. Confirms the supervisor bootstraps state/hub/current to point at the launched binary, then keeps spawning through that symlink.

$ rm -rf /tmp/cc-smoketest/state && mkdir -p /tmp/cc-smoketest/state
$ cloudcode-hub supervise --config ./hub.toml &
hub supervisor starting
    self_exe=/tmp/cc-smoketest/v1.11.0-src/target/release/cloudcode-hub
    spawn_target=/tmp/cc-smoketest/state/hub/current
    config=./hub.toml
hub child spawned   pid=78201
supervisor 78199 (ppid=1) └── cloudcode-hub child 78201 (spawned via state/hub/current → v1.11.0 binary)

Action

$ curl -b cookies http://127.0.0.1:17101/admin/api/me
{"hub_version":"1.11.0","ok":true}

$ pgrep -f "state/hub/current"

$ curl -b cookies -X POST http://127.0.0.1:17101/admin/api/hub/update
{"installed":"v1.11.1","ok":true}

PASS. Supervisor cleanly hot-restarted from v1.11.0 to v1.11.1 through the current symlink, the rollback breadcrumb (previous) was populated correctly, and the user's session loss is recoverable via the SPA's poll-and-reload logic in Layout.tsx.

5. Cross-cutting verifications

  • Hub binary in state/hub/current reports cloudcode-hub 1.11.1 after update
  • Installed binary SHA256 matches the manifest exactly (no truncation, no corruption)
  • Atomic symlink swap: current.tmp renamed onto current; never observed in a half-written state
  • Old binary preserved at previous for the supervisor's 10-failure rollback path
  • Supervisor PID unchanged across update; child PID changed (real re-exec, not in-place swap)
  • Update endpoint returns 202 Accepted with body {"installed":"v1.11.1","ok":true} before exit
  • Hub child exit cycle (~500 ms after 202) lets the response flush to the browser
  • Sandbox cleanup verified: pgrep -f cc-smoketest returns nothing after SIGTERM to the supervisor

6. Out of scope for this report

  • Agent self-update through the hub's UpdateAgent wire message (logic is symmetric — agent's update.rs is the template hub's was mirrored from; coverage gap is the wire dispatch only)
  • Supervisor's 10-consecutive-failure rollback (would require deliberately corrupting a binary; left for a separate fault-injection pass)
  • Linux musl release asset download (host is macOS arm64; CI's release workflow builds + uploads the Linux tarball, but it wasn't downloaded here)
  • Webterm / admin UI visual flows (need a browser fixture; not part of this CLI-level smoke pass)
